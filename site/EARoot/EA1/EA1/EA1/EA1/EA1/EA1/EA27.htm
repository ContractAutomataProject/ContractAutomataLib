Class implementing a label of a Contract Automaton, by extending the super class #lt;tt#gt;Label#lt;/tt#gt;. #lt;br#gt; The content of each label is a list of actions. #lt;br#gt; Contract automata labels can be of three types:#lt;br#gt;<br/><ul>
<li> offer: one action is an offer action and all the others are idle actions,</li><li> request: one action is a request action and all the others are idle actions,</li><li> match: two actions are matching (i.e., one is a request, the other an offer, and h the content is the same) and all the others are idle.</li></ul>
