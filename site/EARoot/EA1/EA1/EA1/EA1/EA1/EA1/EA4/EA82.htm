Class implementing the Model Checking Function. #lt;br#gt; This is implemented by instantiating the #lt;code#gt;CompositionFunction#lt;/code#gt; to the case where two automata are composed: the first is a contract automaton, whilst the second is a generic automaton describing a property. #lt;br#gt; The output is a synchronous product between the contract automaton and the property. #lt;br#gt; <br/>@param #lt;S1#gt; the generic type of the content of states<br/>@param #lt;S#gt; the generic type of states, must be a subtype of #lt;code#gt;State#lt;S1#gt;#lt;/code#gt;<br/>@param #lt;L#gt; the generic type of the labels, must be a subtype of #lt;code#gt;Label#lt;Action#gt;#lt;/code#gt;<br/>@param #lt;T#gt; the generic type of a transitions, must be a subtype of #lt;code#gt;ModalTransition#lt;S1,Action,S,L#gt;#lt;/code#gt;<br/>@param #lt;A#gt; the generic type of the automata, must be a subtype of #lt;code#gt;Automaton#lt;S1,Action,S,T #gt;#lt;/code#gt;  <br/><br/>
