Class implementing the abstract synthesis operator.#lt;br#gt; The synthesis operation is an automatic refinement of an automaton to a refined one where #lt;br#gt; given conditions hold. #lt;br#gt; Contract automata are equipped with two specific conditions: agreement and strong agreement, #lt;br#gt; detailed in the package requirements. #lt;br#gt; The synthesis must also take into account when an action is controllable or uncontrollable. #lt;br#gt; <br/><p> The synthesis is an iterative procedure that at each step i   #lt;br#gt; updates incrementally a set of states Ri containing the bad states, i.e. those states that  #lt;br#gt; cannot prevent a forbidden state to be eventually reached, and refines an automaton Ki. #lt;br#gt; The algorithm starts with an automaton K0 equal to A and a set R0 containing all #lt;br#gt; dangling states in A, where a state is dangling if it cannot be reached from the initial state #lt;br#gt; or cannot reach a final state. At each step i, the algorithm prunes from Kiâˆ’1 in a backwards  #lt;br#gt; fashion transitions with target state in Riâˆ’1 or forbidden source state. The set Ri is obtained  #lt;br#gt; by adding to Riâˆ’1 dangling states in Ki and source states of uncontrollable transitions of A   #lt;br#gt; with target state in Riâˆ’1. When no more updates are possible, the algorithm terminates.  #lt;br#gt; Termination is ensured since A is finite-state and has a finite set of transitions, and at each  #lt;br#gt; step the subsets of its states Ri cannot decrease while the set of its transitions TKi  cannot #lt;br#gt; increase. At its termination the algorithm returns the pair (Ks, Rs). #lt;br#gt; We have that the result is empty, if the initial state of A is in Rs; otherwise, the result is #lt;br#gt; obtained from Ks by removing the states Rs.<br/></p> <br/><p> The abstract synthesis operations generalises the  #lt;br#gt; other synthesis operations by abstracting away the conditions under which a transition is   #lt;br#gt; pruned or a state is deemed bad, thus encapsulating and extrapolating the notion of controllability and  #lt;br#gt; safety. These two conditions, called pruning predicate (Ï†p) and forbidden predicate (Ï†f )   #lt;br#gt; are parameters to be instantiated by the corresponding instance of the synthesis algorithm   #lt;br#gt; (e.g. orchestration or choreography). Predicate Ï†p is used for selecting the transitions to be  #lt;br#gt; pruned. Depending on the specific instance, non-local information about the automaton or  #lt;br#gt; the set of bad states is needed by Ï†p. Therefore, Ï†p takes as input the current transition  #lt;br#gt; to be checked, the automaton, and the set of bad states. If Ï†p evaluates to true, then the  #lt;br#gt; corresponding transition will be pruned. Predicate Ï†f is used for deciding whether a state  #lt;br#gt; becomes bad. The input parameters are the same as Ï†p. However, Ï†f only inspects necessary  #lt;br#gt; transitions. If Ï†f evaluates to true, then the source state is deemed bad and added to    #lt;br#gt; the set of bad states.  #lt;br#gt;<br/></p>  The formal definition is given in Definition 5.1 of:<br/><ul>
